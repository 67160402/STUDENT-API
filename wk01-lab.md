# พิจารณาคำถามต่อไปนี้

1. โค้ดใน index.js มีการตรวจสอบข้อมูลนำเข้า (validation) ก่อนประมวลผลหรือไม่

- มีการตรวจสอบข้อมูลนำเข้า โดยตรวจสอบว่าผู้ใช้ส่ง name และ major มาครบหรือไม่ หากส่งมาไม่ครบจะตอบกลับด้วย Status Code 400

2. ทุก route มีการตอบกลับด้วย Status Code ที่เหมาะสมกับผลลัพธ์หรือไม่

- มีการใช้ Status Code ที่เหมาะสม เช่น 200 OK เมื่อดึงข้อมูล แก้ไขข้อมูล หรือลบข้อมูลสำเร็จ, 201 Created เมื่อเพิ่มข้อมูลนักศึกษาใหม่สำเร็จ, 400 Bad Request เมื่อผู้ใช้ส่งข้อมูลไม่ครบ, 404 Not Found เมื่อไม่พบข้อมูลนักศึกษา

3. โครงสร้างโค้ดในไฟล์เดียวยังอ่านง่ายอยู่หรือไม่ หากมี route เพิ่มขึ้นอีก 10 เส้นทาง จะเกิดปัญหาใดตามมา

- หากรวม Route ทั้งหมดไว้ในไฟล์เดียว ทำให้จำนวน Route เพิ่มขึ้น โค้ดจะยาวและอ่านยาก ค้นหา แก้ไข และดูแลรักษาได้ยาก และหากมีผู้พัฒนาหลายคนจะทำงานได้ยากและเกิดความสับสน ดังนั้นควรแยก Route ออกเป็นไฟล์
