# Peer Check และแก้ไขตามข้อเสนอแนะ

- จับคู่ตรวจสอบงานกับเพื่อนร่วมชั้นเรียน

ชื่อ-นามสกุลผู้ตรวจสอบ (ผู้กรอกแบบฟอร์ม) :

| หัวข้อตรวจสอบ | | ผ่าน / ไม่ผ่าน | | ข้อเสนอแนะ |

URI ใช้คำนามพหูพจน์ ไม่มีคำกริยาปน | ผ่าน |

ใช้ตัวพิมพ์เล็กและขีดกลาง ไม่ใช้ camelCase หรือ underscore | ผ่าน |

HTTP Method สอดคล้องกับการดำเนินการ (เช่น ไม่ใช้ GET เพื่อลบข้อมูล) | ผ่าน |

Status Code ครอบคลุมทั้งกรณีสำเร็จและกรณีผิดพลาด | ผ่าน |

ความสัมพันธ์ระหว่าง resource แสดงด้วย nested path อย่างเหมาะสม | ผ่าน |
