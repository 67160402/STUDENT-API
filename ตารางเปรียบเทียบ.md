# บันทึกผลการเปรียบเทียบ

ตารางสรุปผลการทดลองเปรียบเทียบการใช้งาน REST API และ GraphQL

| หัวข้อเปรียบเทียบ | REST (`/students/:id/full`) | GraphQL |
|---------------------|-----------------------------|----------|
| จำนวนคำขอเพื่อดูข้อมูลนักศึกษา 1 คน | 1 คำขอ (ได้รับข้อมูลทั้งหมดของ Endpoint) | 1 คำขอ (เลือกเฉพาะข้อมูลที่ต้องการ) |
| จำนวนคำขอเพื่อดูข้อมูลนักศึกษาทุกคนพร้อมรายวิชา | มากกว่า 1 คำขอ หากต้องเรียกข้อมูลรายวิชาเพิ่มเติม | 1 คำขอ |
| การเลือกฟิลด์ข้อมูล | ไม่สามารถเลือกฟิลด์ได้ | เลือกฟิลด์ได้ตามต้องการ |
| ปัญหา Over-fetching | มี เช่น `email`, `phone`, `courseIds` | ไม่มี |
| ปริมาณข้อมูลที่ได้รับ | มากกว่าที่ต้องใช้ | ได้เฉพาะข้อมูลที่ร้องขอ |
| ความยืดหยุ่น | ต้องใช้ Endpoint ที่กำหนดไว้ | ปรับ Query ได้ตามความต้องการ |

## สรุปผล

จากการทดลองพบว่า REST และ GraphQL สามารถดึงข้อมูลนักศึกษาและรายวิชาได้เหมือนกัน แต่ GraphQL มีข้อได้เปรียบตรงที่สามารถเลือกเฉพาะฟิลด์ที่ต้องการได้ ทำให้ลดปัญหา Over-fetching และลดปริมาณข้อมูลที่รับส่ง นอกจากนี้ยังสามารถดึงข้อมูลที่เกี่ยวข้องได้ภายในคำขอเดียว ในขณะที่ REST อาจต้องเรียกหลาย Endpoint เมื่อต้องการข้อมูลหลายประเภท จึงเหมาะกับงานที่ไม่ซับซ้อน ส่วน GraphQL เหมาะกับระบบที่ต้องการความยืดหยุ่นและประสิทธิภาพในการดึงข้อมูลมากกว่า
